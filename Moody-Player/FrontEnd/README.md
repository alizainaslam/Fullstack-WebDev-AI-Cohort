# Course Practice Project / Moody-Player

This project was built as part of a course practice assignment.  
The main purpose was to understand full-stack application flow, how the frontend and backend communicate, and how data moves between different layers of the stack.

---

## What I Did

- Built a simple full-stack application  
- Created APIs for uploading songs and getting recommended songs  
- Connected frontend and backend together  
- Implemented mood selection to fetch songs dynamically  
- Displayed song list with an audio player  
- Practiced routing, state management, and API handling  
- Worked with external services like ImageKit for sound hosting
- Use Mongoose DB for store song(s) data

---

## Tech Stack Used

### Frontend
- React  
- Context API  
- Axios  
- Tailwind CSS  

### Backend
- Node.js  
- Express  
- MongoDB  
- ImageKit cloud
- CORS  
- dotenv
